1. Introduction

The Shift to NoSQL

Relational Databases (SQL) have been the industry standard for decades. However, the rise of Big Data exposed critical limitations:

  • Scalability: SQL struggles to scale horizontally across servers.
  • Flexibility: Rigid schemas make it hard to handle unstructured data.

Study Objective

The goal of this assignment is to quantitatively evaluate and compare five NoSQL databases to answer:

"Which database offers the best Execution Time for specific workloads?"

Focusing on Read vs. Write Performance

2. Background of the Study

To understand the results, we first define the core concepts distinguishing NoSQL from SQL.

ACID vs. BASE

SQL (ACID): Strict. Data is always 100% correct immediately (Banks).

NoSQL (BASE): Flexible. Data becomes correct eventually (Social Media likes).

Database Types

  • Key-Value: Redis (RAM-based)
  • Column-Family: Cassandra, HBase
  • Document: MongoDB

CAP Theorem

In distributed systems, you can only guarantee two: Consistency, Availability, or Partition Tolerance.

3. Methodology Overview

The research paper established a controlled benchmarking environment:

  • Benchmark Tool: YCSB (Yahoo! Cloud Serving Benchmark).
  • Dataset: 600,000 Records (1KB each).
  • Hardware: Virtual Machine with 2GB RAM.

Tested Workloads

Workload A: Balanced (50% Read / 50% Update)
Workload C: Read-Heavy (100% Read)
Workload H: Write-Heavy (100% Update)

4. Key Findings & Analysis

Reading Speed (Lower is Better)

Redis 0.5s
MongoDB 16.0s
HBase 28.2s

Analysis: Redis wins easily because it reads from Memory (RAM). HBase is slow here because it reads from Disk.

Writing Speed (Lower is Better)

Redis 1.5s
Cassandra 21.6s
MongoDB 54.7s

Analysis: Cassandra is built for writing logs, so it's fast. MongoDB struggled with locks.

Main Takeaway: In-Memory databases are faster, but Column-Family databases scale better for writes.

7. Conclusion

The experimental evaluation confirms that hardware architecture dictates database performance.

Speed

Redis is unbeatable for raw speed due to In-Memory storage.

Scalability

Cassandra wins for massive write-heavy logs.

Flexibility

MongoDB offers the best balance for complex data structures.

Final Verdict: There is no "Best" database. You must benchmark your specific workload.
